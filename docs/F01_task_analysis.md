# F-01 태스크보드 상세 분석

## 1. F-01 기능 목표

사용자 스펙을 입력받아 구조화된 포트폴리오 JSON을 생성하고, F-02 역량 갭 분석에서 사용할 수 있도록 전달한다.

---

## 2. 세부 태스크

| 태스크 | 설명 | 산출물 | 상태 |
|---|---|---|---|
| 입력 항목 정의 | 희망 직무, 지역, 기술, 자격증, 프로젝트 등 사용자 입력값 정의 | f01_input_output.json | 완료 |
| 출력 항목 정의 | 포트폴리오 요약, 기술 태그, NCS 태그, F-02 전달 데이터 정의 | f01_input_output.json | 완료 |
| 기능 로직 정의 | 입력 → 구조화 → 태그 생성 → F-02 전달 흐름 정의 | F01_portfolio_spec.md | 완료 |
| 테스트 코드 작성 | 샘플 입력값을 포트폴리오 결과로 변환하는 코드 작성 | f01_portfolio_test.py | 완료 |
| NCS API 조사 | 직무/기술을 NCS 능력단위 태그로 변환하기 위한 API 확인 | F01_data_api_check.md | 진행 중 |
| HRD-Net 훈련이력 처리 | 훈련 이력을 실제 API로 받을지 더미로 처리할지 결정 | 협의 필요 | 미정 |
| F-02 연결 확인 | 현재 JSON 구조가 F-02 갭 분석에 충분한지 확인 | 팀 협의 | 협의 필요 |

---

## 3. 현재 확인한 로직

1. 사용자가 희망 직무, 지역, 기술 스택, 자격증, 프로젝트 경험을 입력한다.
2. F-01은 입력값을 JSON 형태로 구조화한다.
3. 기술 스택과 프로젝트 경험을 바탕으로 포트폴리오 요약을 생성한다.
4. 희망 직무와 기술 키워드를 기준으로 NCS 태그를 생성한다.
5. 생성된 결과를 F-02 역량 갭 분석에 전달한다.

---

## 4. 문제 없는지 점검한 부분

- 입력값과 출력값을 JSON으로 분리함
- F-02에서 사용할 수 있도록 `next_step_for_f02` 구조를 따로 둠
- 테스트 코드를 통해 입력값이 포트폴리오 결과로 변환되는 흐름을 확인함
- 폴더 구조를 `docs`, `schemas`, `tests`로 분리함

---

## 5. 협의 필요한 부분

1. NCS 태그를 F-01에서 1차 생성할지, F-02에서 공고 분석과 함께 처리할지 결정 필요
2. HRD-Net 훈련 이력은 실제 API 연동인지, MVP에서는 더미 데이터인지 결정 필요
3. F-02에서 필요한 사용자 스펙 필드가 현재 JSON 구조로 충분한지 확인 필요
4. 포트폴리오 요약을 규칙 기반으로 생성할지, LLM을 사용할지 결정 필요
