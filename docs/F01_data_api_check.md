# F-01 필요 데이터/API 조사

## 1. F-01 기능과 데이터 필요성

F-01은 사용자가 입력한 희망 직무, 지역, 기술 스택, 학력, 자격증, 프로젝트 경험, 훈련 이력을 구조화하여 포트폴리오 JSON을 생성하는 기능이다.

따라서 F-01에서 필요한 데이터는 채용공고 전체 데이터보다 사용자 스펙을 직무 기준으로 정리하기 위한 NCS 기반 데이터가 우선이다.

---

## 2. 우선 활용 데이터/API

### 2-1. 한국고용정보원_워크넷_직무데이터사전 API

사용 목적:
- 사용자 입력 직무와 기술 스택을 NCS 직무/능력단위 태그로 변환

주요 입력:
- serviceKey
- keyword
- 출력건수

주요 출력 컬럼:
- NCS 능력단위명
- NCS 능력단위 정의
- NCS 대분류명
- NCS 대분류코드
- NCS 중분류명
- NCS 중분류코드
- NCS 소분류명
- NCS 소분류코드
- NCS 세분류코드
- NCS 능력단위코드

F-01 활용 방식:
- target.job, profile.skills에서 키워드를 추출한다.
- NCS API 검색 결과를 바탕으로 ncs_tags를 생성한다.
- 생성된 ncs_tags는 F-02 갭 분석에 전달한다.

---

### 2-2. 한국고용정보원_워크넷_직무데이터_표준직무기술서 API

사용 목적:
- 희망 직무의 표준 직무 설명과 요구 역량을 참고하여 포트폴리오 요약 문장을 생성

주요 입력:
- serviceKey
- 수행직무내용
- NCS 능력단위 출력건수

주요 출력 컬럼:
- NCS 능력단위명
- NCS 능력단위 정의
- NCS 대분류명
- NCS 중분류명
- NCS 소분류명
- NCS 세분류코드
- NCS 능력단위코드
- 지식
- 기술
- 태도

F-01 활용 방식:
- target.job을 기준으로 직무 설명을 조회한다.
- 조회 결과를 portfolio_summary, strength_tags, ncs_tags 생성에 참고한다.

---

### 2-3. HRD-Net 훈련과정/훈련이력 관련 데이터

사용 목적:
- 사용자의 훈련 이력을 포트폴리오에 포함
- F-03 훈련 추천 기능과 연결

F-01에서 필요한 컬럼:
- 훈련 과정명
- 훈련 기관명
- 훈련 기간
- 수료 여부
- 관련 기술 키워드
- 관련 NCS 능력단위코드

F-01 활용 방식:
- MVP에서는 사용자가 직접 입력하거나 더미 데이터로 처리한다.
- 추후 실제 API 연동 시 training_history 필드에 저장한다.

---

### 2-4. 워크넷 채용정보 API

사용 목적:
- F-02에서 채용공고와 사용자 포트폴리오를 비교하기 위해 사용

F-01과의 관계:
- F-01은 워크넷 공고를 직접 분석하지 않는다.
- F-01은 F-02가 사용할 수 있도록 target_job, target_region, skills, ncs_tags를 구조화하여 전달한다.

F-01 확인 사항:
- target_job이 채용공고 직무명과 비교 가능한지
- target_region이 채용공고 지역과 매칭 가능한지
- skills, ncs_tags가 공고 요구역량과 비교 가능한지

---

## 3. MVP 데이터 처리 기준

MVP에서는 실제 API 연동 전 아래 방식으로 처리한다.

1. NCS 태그는 일부 직무에 대해 더미 매핑 테이블로 처리
2. 훈련 이력은 사용자가 직접 입력하거나 빈 배열로 처리
3. 워크넷 채용정보는 F-02에서 우선 처리
4. F-01은 사용자 스펙 구조화와 F-02 전달 JSON 생성에 집중

---

## 4. 협의 필요사항

1. NCS 태그를 F-01에서 1차 생성할지, F-02에서 생성할지 결정 필요
2. HRD-Net 훈련 이력은 MVP에서 직접 입력/더미 데이터로 처리해도 되는지 확인 필요
3. F-02에서 필요한 사용자 필드가 현재 JSON 구조로 충분한지 확인 필요
4. F-01에서 실제 API 호출까지 구현할지, 데이터 구조 확인까지만 할지 결정 필요
