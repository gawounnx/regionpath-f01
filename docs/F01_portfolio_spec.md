# F-01 사용자 스펙 입력 및 포트폴리오 JSON 구조화

## 1. 기능 목적

F-01은 사용자의 희망 직무, 희망 지역, 기술 스택, 학력, 자격증, 프로젝트 경험, 활동 이력 등을 입력받아
F-02 역량 갭 분석에서 활용 가능한 표준화된 사용자 포트폴리오 JSON을 생성하는 기능이다.

핵심 목적은 사용자의 흩어진 경험과 스펙을 직무 중심으로 정리하고,
채용공고와 비교 가능한 형태의 데이터로 변환하는 것이다.

---

## 2. 담당 범위

F-01에서 담당하는 범위는 다음과 같다.

- 사용자 기본 정보 입력
- 희망 직무/산업/지역 입력
- 기술 스택, 자격증, 프로젝트, 활동 이력 입력
- 입력값을 포트폴리오 JSON 형태로 구조화
- 필요 시 NCS 태그 후보 임시 생성
- F-02로 전달할 사용자 스펙 JSON 생성

F-01은 채용공고 분석 자체는 담당하지 않는다.
채용공고와 사용자 스펙 비교는 F-02에서 처리한다.

---

## 3. 입력 항목

### 기본 목표 정보

| 필드명 | 설명 | 예시 |
|---|---|---|
| target_job | 희망 직무 | 백엔드 개발자 |
| target_industry | 희망 산업 | IT 서비스 |
| target_region | 희망 근무 지역 | 충청권 |

### 사용자 스펙 정보

| 필드명 | 설명 | 예시 |
|---|---|---|
| education | 학력 | 소프트웨어학과 재학 |
| skills | 보유 기술 | Python, SQL, FastAPI |
| certificates | 자격증 | 정보처리기사 필기 |
| projects | 프로젝트 경험 | AI 포트폴리오 추천 서비스 |
| activities | 활동 이력 | BDAI LLM/RAG 조별활동 |
| training_history | 훈련 이력 | K-디지털 트레이닝 수료 등 |

---

## 4. 출력 항목

| 필드명 | 설명 |
|---|---|
| portfolio_summary | 사용자 포트폴리오 요약 문장 |
| strength_tags | 강점 태그 |
| skill_tags | 기술 태그 |
| ncs_tag_candidates | NCS 직무/역량 태그 후보 |
| next_step_for_f02 | F-02 갭 분석으로 넘길 데이터 |

---

## 5. 처리 로직

1. 사용자가 희망 직무, 지역, 기술, 자격증, 프로젝트 경험을 입력한다.
2. 입력값이 비어 있는지 확인한다.
3. 사용자의 스펙을 직무 중심으로 정리한다.
4. 기술 스택과 프로젝트 경험을 바탕으로 강점을 추출한다.
5. 희망 직무와 입력된 기술 스택을 바탕으로 NCS 태그 후보를 임시 생성한다.
6. 포트폴리오 요약 문장을 생성한다.
7. F-02에서 사용할 수 있도록 JSON 형태로 결과를 반환한다.

---

## 6. F-02 전달 기준

F-02는 채용공고와 사용자 스펙을 비교해야 하므로 아래 필드는 반드시 전달되어야 한다.

- target_job
- target_region
- skills
- certificates
- projects
- skill_tags
- ncs_tag_candidates

---

## 7. 현재 MVP 기준

MVP에서는 완전한 AI 자동화보다 아래 흐름이 정상적으로 작동하는 것을 우선한다.

사용자 입력 → JSON 구조화 → 포트폴리오 요약 생성 → NCS 태그 임시 부여 → F-02 전달

---

## 8. 협의 필요사항

1. NCS 태그를 F-01에서 1차 생성할지, F-02에서 공고 분석 시 처리할지 결정 필요
2. 훈련 이력은 실제 HRD-Net API 연동으로 갈지, MVP에서는 더미 데이터로 처리할지 결정 필요
3. F-02에서 필요한 사용자 스펙 필드가 현재 JSON 구조로 충분한지 확인 필요
4. 프론트 입력 화면은 Streamlit 기준인지 React 기준인지 확인 필요
